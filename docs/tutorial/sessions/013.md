(note series number 16)

Depending on the circumstances in which such a series arises,
a user of the software could choose to either:

-   Leave the protocol template unaltered,
    therfore having such series appropriately flagged as unexpected
    for any datasets in which they are encountered;

-   Include such an acquisition within the protocol template,
    but flag it as an optional acquisition:

    ```json
      ...
      },
      "Acquisition": {
        "is_optional": true,
        "fields": {
          ...
        }
      },
      ...
    ```

-   For a given protocol,
    be permissive of the presence
    of *any* image series without a corresponding entry in the template
    (ie. only check for conformity of a subset of the acquired data):

    ```json
    {
      "GENERAL": {
        "allow_extra": true
      },
      ...
    ```
